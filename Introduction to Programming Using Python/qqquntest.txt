              <div style="font-family:'Helvetica Neue'">

              <div style="font-family:'Helvetica Neue'">

               <b>

               <u>
                做好复习
               </u>
               也是在做预习，一方面是可以类比推理，另一方面也是递进的关系 ）

               </b>

              <div style="font-family:'Helvetica Neue'">
               [关于进步收敛]
              </div>
              <div style="font-family:'Helvetica Neue'">
               课程笔记：
              </div>
              <ul style="font-family:'Helvetica Neue'">
               <li>
                <span style="font-size:11px">
                 <span style="color:rgb(1, 25, 147)">
                  犯过的错误不要再犯，不能变着花样再犯
                 </span>
                 （收敛速度来快速成长）
                </span>
                <ul>
                 <li>
                  <span style="font-size:11px">
                   找到那个核心的错误点，用持续行动来修正（因为没有get，没有触类旁通的把该改的都改了）
                  </span>
                 </li>
                </ul>
               </li>
               <li>
                <span style="font-size:11px">
                 <span style="color:rgb(1, 25, 147)">
                  如果你能确保每次不犯旧错误，你的进步是收敛的
                 </span>
                </span>
                <ul>
                 <li>
                  <span style="font-size:11px">
                   否则你的系统是耗散的，就像漏风的强（5个点，收起来像聚光灯）
                  </span>
                 </li>
                </ul>
               </li>
              </ul>
              <div style="font-family:'Helvetica Neue'">
               <br/>
              </div>
              <div style="font-family:'Helvetica Neue'">
               我的理解：
              </div>
              <div style="font-family:'Helvetica Neue'">
               <span style="color:rgb(255, 47, 146)">
                在行动的过程中暴露了问题，然后我们集中火力来解决这些问题，下次交付的时候要着重检查，不犯同样的错误。这样做就相当于你在给自己的系统打补丁。你的交付越来越完善，那么就是在进步的，就和英语朗读一样，发现盲点然后改过来，慢慢的问题原来越少，无限趋近于最好。
               </span>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <br/>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <br/>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <span style="font-size:18px">
                补充了什么：
               </span>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <u>
                自己在生活中有体会，但是听课之后更清晰的点
               </u>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <span style="color:rgb(1, 25, 147)">
                [关于输出倒逼输入]
                <b>
                 过了一个基础线，就可以开始输出再输入
                </b>
               </span>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <span style="color:rgb(255, 47, 146)">
                比如在吉他的学习中，跟着老师入门，把基本的按弦方法乐理知识认谱等基础打捞，然后就可以自己不断地去练习，想学什么曲子再push自己去学，也更有动力。这里的动力是可以简单的输出之后带来的自信。其实也是一种
               </span>
               <span style="color:rgb(255, 47, 146)">
                结果导向，看要输出什么然后决定去输入什么。
               </span>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <br/>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <span style="color:rgb(1, 25, 147)">
                [影响行动力的因素]
                <b>
                 对于你要做的事情，排上足够高的优先级，不断升级，不断取舍（做事不要挑容易的做）
                </b>
               </span>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <span style="color:rgb(255, 47, 146)">
                现在的情况就是会先挑容易的做，像青蛙理论说的是早上起来先完成最难的一个任务，这样一天会觉得比较轻松。具体体现就是第1题中发现的问题，
                <b>
                 每天给学业保证一定的时间支出来完成对应任务
                </b>
                这点做的很不好，大概就是优先级排列出问题了。把学业的任务拖到最后完成，因为觉得难，所以先做其他的语言啊成长会的任务，熟练并且有趣，理所应当的把反馈比较慢的学习任务往后推然后没有时间就一直拖到ddl。
               </span>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <br/>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <span style="-evernote-highlight:true">
                不断锻炼行动力是不是就是优先级低的也可以做到，从上好到下？？
               </span>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <br/>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <br/>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <span style="font-size:18px">
                改变了什么：
               </span>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <u>
                观点触发思考并且决定应用到实践中
               </u>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <span style="color:rgb(1, 25, 147)">
                [影响行动力的因素]
                <b>
                 即使有扰动，也没有什么大不了的，不要失控就行（把计划写满是心理补偿，我现在这么烂！）
                </b>
               </span>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <span style="color:rgb(255, 47, 146)">
                我现在的问题是没有办法保证上学的时候经历充沛，特别是这几天放假之后每天早上四个半小时的课很不适应，效率很低。就像第1题中的
                <b>
                 如何高效地听讲
                </b>
                （保证睡眠时间）这点做的很不好一样，每天的任务都排的很满，因为觉得自己还差好多，恨不得每天不睡觉地翻倍完成任务。但是事实就是，安排那么多其实也没完成了，反而把自己弄的很累，第二天效率很低。
               </span>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <span style="color:rgb(255, 47, 146)">
                所以不是要贪多，要集中火力做重要的几件事。这里的优先级战略调整或者说是取舍就不是说不勇敢，不上进的问题了，而是知道自己还有很多地方做的不够好，但是有平心静气慢慢来做的魄力。就是被括号里的那句话打脸打的很疼，一直以来把自己弄的很忙很累，好像很努力的样子，但是其实同时做的几件事都没有做的很好。
               </span>
              </div>
              <div style="font-family:'Helvetica Neue'">
               <span style="color:rgb(255, 47, 146)">
                好好想想，现在的重心在哪个方面，虽然说实习毕业申请研究生几件事必须同时做，但是细化一下每一项的内容，其实也不是那么的繁琐，所以查好ddl，决定好的就去申请，材料准备齐全，安心备考等待结果。学习上的事情不能再拖了，非要等到考试才知道之前都没有好好学吗。回家先回顾当天的课程，然后完成作业，中间可以穿插填表申请之类的事情，控制好时间，先觉得学业这个主要矛盾，分清主次，不要把做了别的事当做做不好学习这件事的借口。
               </span>
              </div>
              <div style="font-family:'Helvetica Neue'">
